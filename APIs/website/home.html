SMART TRANSPORTATION

To decrease commute time as well as enable multi-vehicle control by a single driver using GPS and Ultrasonic sensors using the concepts of IoT and Machine Learning.

Current issues with autonomous self-driving cars

Sometimes drivers have to make instant decisions when they are faced with obstacles on the road. It’s pretty easy to program a car to make an emergency stop if it detects an object in the middle of the road.

‘​‌‘​‌

The crucial flaw of current self-driving cars is that they will always need human involvement. Currently, no self driving car can completely drive on its own.

‘​‌‘​‌

Bad weather can be a real problem when you consider that self-driving cars use cameras to track the lines on the pavement. While humans are capable of simply following the natural curve of the road, driverless cars aren’t quite there yet.

‘​‌‘​‌

One of the most difficult aspects of transitioning to self-driving cars is the legal gray area that exists. Even a car that technically functions perfectly according to road laws and mechanical performance could still perform an operation that causes negative results.

Technologies used

Keras

Used in our machine learning model to identify the leading car.

NodeMCU

Used to establish an IoT communication between client and server.

Arduino

Used Arduino as an interface between the software and the hardware.

Bluetooth

Used to establish a wireless connection between the vehicles.

Flask

Used it as a backend for the website and stored it in a Database.

HTML, JavaScript & CSS

Used these to construct the front end for the model.

Applications

Greatly decreasing commute time across major metropolitan areas.

Reducing costs for cargo transportation by allowing a single driver to control more than one vehicle.

Allowing autonomous vehicles to drive multiple non-autonomous vehicles at the same time.
